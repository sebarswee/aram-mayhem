# H5肉鸽游戏设计文档

## 概述

**项目名称**：暂定《技能乱斗》(Skill Brawl)

**游戏类型**：H5即时战斗肉鸽割草游戏

**核心概念**：融合LOL大乱斗的随机技能机制 + TFT海克斯符文选择 + 吸血鬼幸存者的割草爽感

**目标平台**：H5网页（后期可打包微信/抖音小游戏）

---

## 核心玩法循环

```
┌─────────────────────────────────────────────────────────┐
│                      游戏开始                            │
│                         ↓                               │
│            【初始Roll】3次机会选择技能组合               │
│                  (可锁定部分技能重roll)                  │
│                         ↓                               │
│                  【战斗阶段】                            │
│         ┌──────────────┴──────────────┐                │
│         ↓                              ↓               │
│   自动释放技能                    玩家控制走位          │
│   (3小技能+1大招)               (虚拟摇杆/键盘)         │
│         ↓                              ↓               │
│         └──────────────┬──────────────┘                │
│                        ↓                               │
│                  击杀敌人 → 获得经验                    │
│                        ↓                               │
│                  升级 → 选择符文(三选一)                │
│                        ↓                               │
│              每10波遭遇BOSS                            │
│                        ↓                               │
│                  死亡 or 无尽继续                       │
│                        ↓                               │
│              【结算】获得荣耀点数                       │
│                        ↓                               │
│         局外解锁新技能/符文加入池                       │
└─────────────────────────────────────────────────────────┘
```

## 技能系统

### 技能池规模

| 技能类型 | 初始数量 | 后期扩展 |
|----------|----------|----------|
| 小技能(Q/W/E类) | 45个 | 60+ |
| 终极技能(R类) | 20个 | 30+ |
| 被动技能 | 15个 | 25+ |

### 技能分类与标签系统

**元素标签**：
- 火焰 - 燃烧、爆炸、AOE
- 冰霜 - 减速、冰冻、控制
- 雷电 - 连锁、麻痹、爆发
- 物理 - 穿刺、流血、暴击
- 暗影 - 吸血、诅咒、持续伤害
- 神圣 - 治疗、护盾、增益

**类型标签**：
- 投射物 - 直线飞行的弹道技能
- 范围 - AOE效果
- 位移 - 冲刺、闪现、击退
- 召唤 - 召唤物辅助战斗
- 增益 - 自我强化
- 控制 - 眩晕、减速、击飞

**范围标签**：
- 近战 - 角色周围小范围
- 中程 - 中等射程
- 远程 - 长距离攻击

### 技能数据结构

```typescript
interface Skill {
  id: string;
  name: string;
  description: string;
  type: 'basic' | 'ultimate' | 'passive';

  // 标签系统
  elements: Element[];
  categories: Category[];
  range: Range;

  // 技能参数
  cooldown: number;        // 冷却时间(秒)
  damage: number;          // 基础伤害
  rangeValue: number;      // 作用范围/距离

  // 特效
  effects: SkillEffect[];

  // 视觉
  sprite: string;
  animation: string;
  particles: ParticleConfig[];

  // 稀有度(仅大招)
  rarity?: 'common' | 'rare' | 'legendary';
}

interface SkillEffect {
  type: 'damage' | 'burn' | 'freeze' | 'stun' | 'knockback' | 'heal' | 'shield';
  value: number;
  duration?: number;
}
```

### 技能组合机制

**1. 元素叠加效果**

当玩家拥有多个相同元素的技能时，触发额外效果：

| 元素 | 2个技能 | 3个技能 | 4个技能 |
|------|---------|---------|---------|
| 火焰 | +10%燃烧伤害 | 燃烧范围+50% | 燃烧可暴击 |
| 冰霜 | 冻结时间+0.5s | 冻结范围+30% | 冻结后额外伤害 |
| 雷电 | 连锁+1目标 | 麻痹时间+0.3s | 连锁伤害衰减减少 |
| 物理 | +5%暴击率 | 流血伤害+50% | 暴击时触发流血 |
| 暗影 | +5%吸血 | 诅咒持续时间+2s | 击杀诅咒敌人回血 |
| 神圣 | +10%治疗量 | 护盾+20% | 满血时治疗转护盾 |

**2. 技能联动**

部分技能设计时考虑combo关系：

- **击飞类技能** → 后续技能对空中敌人伤害+50%
- **眩晕类技能** → 对眩晕目标暴击率+30%
- **减速类技能** → 对减速目标有概率冻结
- **燃烧类技能** → 对燃烧目标击杀时爆炸

**3. 符文增强**

符文可以强化特定标签的技能（详见符文系统）

### 自动释放AI

**目标选择优先级**：
1. 最近敌人
2. 血量最低敌人
3. 精英/BOSS
4. 随机

**释放逻辑**：
- 技能冷却完成 → 检测范围内是否有目标
- 有目标 → 释放技能
- 无目标 → 等待

**优化**：
- 大招优先在敌人密集区域释放
- 位移技能用于躲避危险或追击

## 符文系统

### 符文类型与权重

| 类型 | 出现权重 | 稀有度倾向 | 示例 |
|------|----------|------------|------|
| 属性增强 | 40% | 白/绿 | +10%攻击力、+15%移速 |
| 技能强化 | 25% | 绿/蓝 | 技能穿透+1、技能范围+20% |
| 被动效果 | 20% | 蓝/紫 | 击杀回血、受伤反伤 |
| 新增技能 | 10% | 紫 | 获得额外小技能 |
| 特殊机制 | 5% | 金(传说) | 技能暴击、双持技能 |

### 符文稀有度

| 稀有度 | 颜色 | 出现概率 | 效果强度 |
|--------|------|----------|----------|
| 普通 | 白 | 50% | 基础数值加成 |
| 稀有 | 绿 | 30% | 中等加成 |
| 史诗 | 蓝 | 15% | 较强效果 |
| 传说 | 紫 | 4% | 强力效果 |
| 神话 | 金 | 1% | 改变玩法的效果 |

### 符文数据结构

```typescript
interface Rune {
  id: string;
  name: string;
  description: string;
  rarity: Rarity;
  type: RuneType;

  // 效果
  effects: RuneEffect[];

  // 互斥组
  exclusiveGroup?: string;  // 同组符文互斥

  // 升级
  maxLevel: number;
  upgradeFrom?: string;     // 升级来源符文ID
}

interface RuneEffect {
  type: 'stat_boost' | 'skill_enhance' | 'passive' | 'new_skill' | 'special';
  target?: 'all' | 'element' | 'category';  // 作用目标
  targetValue?: string;                      // 具体标签值
  value: number;
  isPercent: boolean;
}
```

### 符文互斥机制

**互斥组设计**：

| 互斥组 | 符文示例 | 说明 |
|--------|----------|------|
| 元素专精 | 火焰之心 / 冰霜之心 / 雷电之心 | 每个元素专精互斥 |
| 攻击模式 | 狂战士(高攻低防) / 守护者(高防低攻) | 玩法风格互斥 |
| 技能强化 | 技能穿透 / 技能爆炸 | 同类强化互斥 |

**非互斥**：
- 所有属性增强符文可自由叠加
- 不同类型的被动效果可共存

### 符文升级机制

**叠加升级规则**：
- 相同符文拾取2个 → 合成升级1级
- 最高3级
- 已满级符文不再出现

**示例**：
```
攻击力I (+10%) × 2 → 攻击力II (+18%)
攻击力II (+18%) × 2 → 攻击力III (+25%)
```

### 升级选择流程

```
升级触发
    ↓
暂停游戏
    ↓
生成3个候选符文（基于权重+稀有度）
    ↓
玩家选择1个
    ↓
应用效果（检查互斥、处理升级）
    ↓
继续游戏
```

## 初始Roll技能机制

### Roll流程

```
游戏开始
    ↓
显示Roll界面（剩余次数: 3）
    ↓
随机生成一套技能（3小技能 + 1大招）
    ↓
┌─────────────────────────────────────┐
│  [技能1] [技能2] [技能3] [大招]      │
│   [锁定]  [锁定]  [锁定]             │
│                                     │
│  [重新Roll] [确认这套技能]          │
└─────────────────────────────────────┘
    ↓
玩家选择: 重roll or 确认
    ↓
重roll: 消耗1次机会，锁定的技能保留
确认: 进入战斗
    ↓
机会用尽: 强制使用当前技能组
```

### 大招稀有度

| 稀有度 | 概率 | 特点 |
|--------|------|------|
| 普通 | 70% | 基础大招 |
| 稀有 | 25% | 效果更强/范围更大 |
| 传说 | 5% | 独特效果/多段伤害 |

### 锁定机制

- 每个技能可单独锁定
- 锁定后重roll时，该技能保留
- 策略性：锁定满意技能，赌更好的组合

## 敌人系统

### 敌人分类

| 类型 | 数量 | 特点 |
|------|------|------|
| 普通怪 | 8种 | 低血量、数量多、割草感 |
| 精英怪 | 5种 | 中血量、特殊技能、掉落更多经验 |
| BOSS | 3种 | 高血量、多阶段、独特技能 |

### 敌人数据结构

```typescript
interface Enemy {
  id: string;
  name: string;
  type: 'normal' | 'elite' | 'boss';

  // 属性
  hp: number;
  damage: number;
  speed: number;

  // 行为
  behavior: 'chase' | 'ranged' | 'summon' | 'teleport';

  // 掉落
  expValue: number;

  // 特殊能力
  abilities: EnemyAbility[];

  // 视觉
  sprite: string;
  animations: AnimationConfig;
}

interface EnemyAbility {
  type: 'charge' | 'shoot' | 'summon' | 'shield' | 'heal';
  cooldown: number;
  params: Record<string, any>;
}
```

### 波次生成规则

| 波次 | 敌人数量 | 敌人类型 | 精英概率 | BOSS |
|------|----------|----------|----------|------|
| 1-3 | 20-40 | 普通怪×1种 | 0% | 无 |
| 4-6 | 40-60 | 普通怪×2种 | 5% | 无 |
| 7-9 | 60-80 | 普通怪×3种 | 10% | 无 |
| 10 | 30+精英×2 | 普通怪+精英 | 30% | BOSS |
| 11-19 | 递增 | 多种普通怪 | 15% | 无 |
| 20 | 40+精英×3 | 全类型 | 40% | 强BOSS |
| ... | 递增 | 全类型 | 递增 | 每10波 |

### 敌人AI行为

**普通怪**：
- 简单追踪玩家
- 接触玩家造成伤害

**精英怪**：
- 追踪 + 特殊技能（冲锋/远程攻击）
- 更聪明的目标选择

**BOSS**：
- 多阶段技能循环
- 特殊机制（召唤小怪/护盾/狂暴）
- 明显的技能前摇提示

### 经验获取

| 来源 | 经验值 |
|------|--------|
| 普通怪 | 1-3 |
| 精英怪 | 10-20 |
| BOSS | 50-100 |

### 升级曲线

| 等级 | 所需经验 | 累计经验 |
|------|----------|----------|
| 1→2 | 10 | 10 |
| 2→3 | 15 | 25 |
| 3→4 | 22 | 47 |
| 4→5 | 30 | 77 |
| ... | 递增 | ... |
| 10→11 | 100 | 385 |
| ... | 继续 | ... |

**公式**：`expNeeded = floor(10 * level^1.5)`

### 升级触发

```
击杀敌人 → 获得经验 → 经验条满
    ↓
等级+1 → 暂停游戏 → 弹出符文选择
    ↓
选择符文 → 应用效果 → 继续游戏
```

## 局外成长系统

### 荣耀点数

- 每局结束根据表现获得
- 计算公式：`荣耀点数 = 击杀数 × 0.1 + 存活波次 × 5 + BOSS击杀 × 20`

### 解锁系统

**解锁内容**：

| 类型 | 解锁项 | 荣耀点数 |
|------|--------|----------|
| 新技能 | 额外小技能加入技能池 | 100/个 |
| 新符文 | 新符文加入符文池 | 50/个 |
| 新角色 | 不同初始属性/外观 | 500/个 |
| 新模式 | 无尽挑战模式 | 1000 |

**注意**：解锁的是"加入池"，不是"直接拥有"，保持肉鸽的随机性

## 数值设计原则

### 玩家成长曲线

- 前期（1-10级）：快速成长，每级都有明显提升
- 中期（11-30级）：稳定成长，依赖符文组合
- 后期（31+级）：成长放缓，考验操作与build质量

### 敌人难度曲线

- 波次1-10：熟悉期，敌人伤害低、数量少
- 波次11-20：挑战期，精英增多，BOSS战
- 波次21+：挑战极限，考验玩家build上限

### 平衡原则

- 避免绝对最优解，鼓励多样化build
- 稀有/传说符文强但不必须
- 技能组合收益明显但不强制

## 总结

《技能乱斗》是一款融合了：
- **LOL大乱斗**的随机技能机制
- **TFT**的海克斯符文选择
- **吸血鬼幸存者**的割草爽感

核心体验：每局都是新的build探索，通过技能组合 + 符文强化，创造独特的战斗风格。

技术实现：Phaser 3 + TypeScript，模块化架构，便于扩展维护。

开发策略：MVP优先核心循环，迭代扩展内容。
