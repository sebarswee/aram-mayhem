# 技能系统重构设计文档

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 重构游戏核心系统，实现8元素羁绊系统、超级割草体验、技能进化系统、食物掉落生存机制、无限地图

**Architecture:**
- 数据驱动架构，所有配置从文件读取
- 元素系统独立模块，管理羁绊触发
- 技能系统支持自动释放和进化分支
- 多端适配（PC/移动端）

**Tech Stack:** Phaser 3, TypeScript, Vite

## Global Constraints

- 使用Phaser 3的RESIZE scale mode实现多端适配
- 所有数据（元素、技能、怪物、掉落）必须从配置文件读取，支持后期扩展
- 基础技能自动释放，大招手动触发
- 无限地图设计，敌人生成围绕玩家位置
- 超级割草模式：大量敌人（30-100个同时在场），技能高频触发

## 3. 强化系统

### 3.1 技能升级机制

**升级方式：** 收集经验升级时，随机出现强化选项

| 选项类型 | 出现概率 | 说明 |
|----------|----------|------|
| 技能强化 | 60% | 强化已有技能（选择某个技能升级） |
| 新技能 | 25% | 学习新技能（技能槽未满时） |
| 属性提升 | 15% | 提升玩家基础属性 |

### 3.2 技能强化树

**每个技能有5个等级：**

| 等级 | 解锁内容 |
|------|----------|
| Lv1 | 基础形态 |
| Lv2 | 二选一强化（如：伤害+30% 或 范围+30%） |
| Lv3 | 二选一强化 |
| Lv4 | 二选一强化 |
| Lv5 | 三选一进化分支（质变） |

### 4.3 怪物配置结构

```typescript
interface EnemyConfig {
  id: string;                    // 唯一ID
  name: string;                  // 显示名称
  type: 'normal' | 'elite' | 'boss';  // 类型
  element: Element;              // 元素属性
  hp: number;                    // 基础HP
  damage: number;                // 基础伤害
  speed: number;                 // 移动速度
  expValue: number;              // 经验值
  color: number;                 // 颜色（临时用）
  abilities: EnemyAbility[];     // 特殊能力
}

interface EnemyAbility {
  type: string;                  // 能力类型
  trigger: 'attack' | 'death' | 'passive';  // 触发时机
  params: Record<string, any>;   // 参数
}
```

| Lv10+ | 每10级经验需求翻倍 | 选择强化 |
